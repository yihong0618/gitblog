## 重启 tg_bot_collections

那时候还没有🦞。

在不是所有人都有机会用大模型的时候我做了这个项目 -> [tg_bot_collections](https://github.com/yihong0618/tg_bot_collections) 这是一系列当时为了方便群友使用大模型，使用有趣的新东西的 bot 集合，当时花了很多心思，为了如何更方便的给群友们用，今天回头看，很多设计都很领先，当然还是一直感谢 frost 他每次都能把的不是那么好的代码变得工业化漂亮几十倍，在现在每个 LLM 都能轻松的再他重构的架构下轻松加上新的 bot command.

不过这次没有再开源了，时代变了。

我用一周的时间做了过去两年没有做的 todo list, 加了 check 功能加了remind 功能，甚至因为过年我加了 /fire 和 /happy 功能，你能在 telegram 里放一个烟花，这个功能也是当时我们发现能用 telegram 的 edit 来实现 stream 一样，非常有趣。当然，代码 80% 都是 LLM codex 写的我只负责设计，为啥还剩了 20%？ 那些不是大模型完不成的东西，只是我想写一些代码。

这次更新最感动的一句话是，当我加了 RAG 功能后，朋友的一句话。`这是我第一次看到 RAG 真正的应用`.

当然这次重启最牛逼的是，给 bub 的诞生做了一点点贡献。

## 来手写一些代码

说到我想手写一些代码，什么时候我觉得我不再需要写代码了呢？去年我花了非常多的时间试图为了证明我比大模型强一点点，去给各种 DB 贡献，做优化，去给 cpython 贡献去解决最难的问题。是的结果是好的，大模型在进化我也在进化，给 cpython 解决了一个几个月没人去解决的问题后（在 gray 和朋友的共同的帮助下）我觉得我还可以，但两件事之后我觉得现在已经没有大模型解决不了的问题了：

1. 在 xuanwo 的一次分享后我问题了一个问题，我说现在你遇到过大模型解决不了的难题么？xuanwo 的回答是有，但是原因不是大模型能力不够，而是他当时没有给大模型足够的 context
2. 我一直 watching cpython 项目，亲眼见证了一个大哥，一直用 LLM 给 cpython 提代码，reviewer 每次给他提意见他再用 LLM 去改，几轮下来之后 PR 就被关了，他大概提了 10 几个 PR 只合了一个，但是 Opus 4.5 出来以后他的 PR 质量大幅度的上升，已经基本都会被合并了，我看到了这个现象之后就很少再去主动解决 issue 提 patch 了，因为大部分只需要一条 prompt 了(P.S.我遇到了我需要的还是会提的)。

没有跑题: 于是我在想如何才能保证每天手写一些代码呢？ LeetCode easy 似乎是最好的方式，它足够简单，又足够有趣，每天花费不了太多时间，还能让我们有一些手写代码的手感，于是我就在频道每天随机一道 easy 题和大家来一起手写，因为足够简单也没有 cheat 可言，没想到所有人面试深恶痛绝的 LeetCode 成了手写代码的替代。

## 转变

这两年在博客里好像也写过我的心态发生了不少的改变，古人真的挺厉害，越接近 40 越发的觉得自己的困惑确实少了。

变得圆滑世故也好，变得不再少年磨平棱角也罢，对曾经那些深恶痛绝的事儿越发的觉得无所谓，也不会面红耳赤的去争个对错。也越来越少的觉得对面是个傻逼拉黑了事儿，
做的越来越多的是反思自己有的时候自己做了不对的事儿朋友指出来我会主动去反思自己哪些地方做的不那么好。也会试着去解决自己父母多年不太好的关系，去做其中的调和剂。虽然生活没有变得那么好，但是似乎给家人朋友们带来了一些开心的东西。

也学着去只在意自己应该在意的人。
> 你无法让所有人都喜欢你

但是这个月有两个朋友我一直想努力去帮忙没有得到太好的结果有些伤心失落，按照我的经验以前的我可能会说一些自己的想法诸如 xxx.帮朋友权衡利弊，但是现在的我更珍惜的是我做为朋友能帮些什么，我能做些什么，于是我觉得 follow your heart 是这里对朋友最好的回复。
